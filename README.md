# Deque README


## 项目概述

在小作业中，大家已经使用过STL的 `std::deque` 的头尾插入删除功能，实际上，`std::deque` 也支持类似普通数组那样的随机下标访问。

在这次大作业中，你需要使用分块链表（Unrolled Linked List）数据结构实现高效的双端队列（Deque），支持快速随机访问与动态容量调整。所需要实现的接口在给定的文件中。

## 核心要求

分块链表就是对存储元素进行分块，以加速随机访问等操作。理想情况下，每一块的大小在 $O(\sqrt n)$ 量级。一种保证方法是在块过大时分成两块，相邻两块均很小时合成一块。在具体实现中，每一
块内部的储存方式和所有块的储存方式均可使用链表或数组。 你需要保证头尾插入和删除的均摊
复杂度是 $O(1)$，随机插入、删除和查询的复杂度是最坏 $O(\sqrt n)$。

这次的大作业不允许使用STL容器，但你可以使用你上次写的双向链表，直接粘贴进 `src.hpp` 即可。这次任务的几乎所有测试都在下发的文件中，其中带有 memcheck 的测试仅是其原始测试数据更少
的版本，可用作检查内存泄漏和内存错误。

我们在此提供两种可以考虑的实现方式：一种是链表套链表，另一种的链表套循环数组，一般来说后一种实现的效率会更高一些。**注：你的实现必须包括动态调整块长的策略，不允许固定块长**

## 策略说明

本实现使用“链表套分块数组”的结构：外层是双向链表维护块顺序，内层每个块是连续指针数组，块内元素在区间 `[start, end)` 有效。

### 1. 动态块长策略

块容量不是固定常数，而是随当前规模 `n=size()` 动态调整：

$$
B(n)=\text{clamp}(2\sqrt{n},\ 64,\ 2048)
$$

实现中额外保证容量为偶数，便于中点分裂。这样在小数据时避免频繁分块，在大数据时避免单块过大。

### 2. 分裂策略（块过大）

当在某块插入，且该块两端都没有空位、块已满时：

1. 在中点 `mid=cap/2` 处分裂。
2. 右半元素搬到新块，新块插入原块后方。
3. 根据插入位置落在左块或右块继续完成插入。

分裂后两个块的大小都保持在同一量级，避免出现线性退化的“巨块”。

### 3. 合并与借位策略（块过小）

删除后调用统一维护函数：

1. 设下界 `low=max(B(n)/2, 16)`。
2. 若当前块大小 `< low`，优先尝试与右邻块合并。
3. 若不能合并，再尝试“前驱+当前”合并。
4. 若仍不能合并，则向右邻/左邻借元素做局部重平衡（邻块大于下界时才借）。

这样可保证块大小长期维持在 `Theta(B(n))`，不会积累大量很小的碎块。

### 4. 块内重排（recenter）

当块未满但仅一侧无空间时，先把块内元素整体居中，再尝试头/尾插入。这样可以避免不必要的新块分配，降低常数开销。

### 5. 复杂度说明

记块大小量级为 `B=Theta(\sqrt n)`，块数约为 `n/B=Theta(\sqrt n)`：

1. 头尾插入/删除：均摊 `O(1)`。
2. 随机访问：跨块移动最多 `O(\sqrt n)`。
3. 随机插入/删除：定位 + 局部搬移/分裂/合并，最坏 `O(\sqrt n)`。
4. 分裂、合并、借位都只发生在相邻块，单次代价 `O(B)=O(\sqrt n)`。

因此整体满足题目要求的复杂度上界。

### 6. 边界条件与异常语义

#### 6.1 空容器边界

1. `pop_front/pop_back/front/back/erase` 在空容器上触发 `container_is_empty`。
2. `begin()==end()` 与 `cbegin()==cend()` 在空容器成立。

#### 6.2 下标与访问边界

1. `at(pos)` 与 `operator[](pos)` 在 `pos>=size()` 时抛出 `index_out_of_bound`。
2. 迭代器解引用时若指向 `end()` 或无效位置，抛出 `invalid_iterator`。

#### 6.3 插入删除位置边界

1. `insert(end(), x)` 合法，等价尾插。
2. 非本容器迭代器、或结构不匹配位置的迭代器，插入/删除均抛 `invalid_iterator`。
3. `erase` 删除最后一个元素时返回 `end()`；否则返回后继元素位置。

#### 6.4 块结构边界

1. 删除导致块空时，立即移除该块，避免悬空空块。
2. 合并时若两块总元素数超过目标容量则拒绝合并，转为借位。
3. 合并构造新块时按总元素量重新居中设置 `[start,end)`，避免数据越界或丢失。

#### 6.5 迭代器边界

1. `--end()` 在非空容器合法，定位到最后一个元素。
2. `begin()` 再前移、`end()` 再后移等非法移动抛 `invalid_iterator`。
3. const 迭代器与普通迭代器的后置自增/自减返回值类型保持一致，避免未定义行为。

以上策略保证了正确性、鲁棒性与复杂度目标的统一。
